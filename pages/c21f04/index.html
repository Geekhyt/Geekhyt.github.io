眼里要有光！分享一些我收藏夹里的链接

问渠哪得清如许，为有源头活水来。

收到了大家的催更，深表歉意。最近工作十分忙碌，连午睡时间也牺牲给了来面试的同学，放假休息了两天给身体充满了电。

这就来给大家分享一些最近的感悟，虽然没时间更新原创，不过利用上下班的路上看完了一本书。

每周最低阅读 3 小时的习惯从去年就开始了，把这些独处的碎片化时间利用起来，收益比想象中要大很多。

去年一整年居然看完了 32 本书，感兴趣的同学可以稍移玉步到我的 《2020年终总结》。

拉回来，这本书就是古典写的《跃迁：成为高手的技术》，书中提到现在我们获取的知识绝大多数是二三四手信息，因为很多人已经失去了鉴别一手信息的能力，这也是我们认知效率低下的原因。书中论述到：

  1. 一手信息：知识的源头，如：论文、学术期刊、行业最新数据报告、行业大牛的最新沟通和思考。
  2. 二手信息：忠实转述，有清晰的论据和出处。如维基百科、讲述底层逻辑、思考质量比较高、略微难懂的书和文章，如：《国富论》、各行业领军人物、行业大牛推荐的书单，以及优质的公众号文章。
  3. 三手信息：畅销书，更方便传播，方便大众理解，加入大量案例、故事以及不精确的概念。
  4. 四手信息：根据这些畅销书和理论，大部分人写了基于个人体验的鸡汤，加入了太多个人的故事或者情绪因素，如：《我是如何一个小时挣到 200 万的》。

回到我们作为一名前端工程师的视野，前端知识的源头便是 Web 标准，毫无疑问它是构成前端技术的原力，每一个前端工程师都要认真学习。

Web 标准由多家标准组织制定和规划，包括 W3C、IETF、Ecma、WHATWG 等，相关链接整理如下：

# 01 Web 标准组织

# W3C

World Wide Web Consortium(万维网联盟)，于 1994 年成立，是 Web 标准的主要制定者。

  • W3C

# IETF

Internet Engineering Task Force(互联网工程任务组)，于 1986 年成立，TCP/IP 协议就是这个组织标准化的。

IETF 成立了 HTTP Working Group (HTTP WG) ，基于 HTTP/0.9 制定了 HTTP/1.0 等一系列规范。

# HTTP/0.9

  • HTTP/0.9

# HTTP/1.0

  • RFC1945

# HTTP/1.1

  • RFC2068
  • RFC2616
  • RFC7230-RFC7235

# TLS 1.3

  • RFC8446

# HTTP/2

  • RFC7540

# Ecma

Ecma International 于 1961 年成立，C#、Dart、JavaScript 都由其进行标准化，其中，TC39 负责 ECMA-262 规范的制定。

  • TC39
  • ECMA-262

# WHATWG

Web Hypertext Application Technology Working Group (Web超文本应用技术工作组)，与 W3C 合作制定 HTML 和 DOM 的标准。

  • WHATWG

除了公司内部的技术分享，参与技术大会也是绝佳的知识获取方式，可以开拓视野、和行业大牛交流学习行业内的最佳实践，也符合书中提出的 联机学习。
